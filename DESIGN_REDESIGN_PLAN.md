# План редизайна SmoothScrollModern

## Цель

Сделать интерфейс приложения логичным, понятным и визуально цельным в стиле современных Windows-приложений: нормальная верхняя панель окна, левая навигация, спокойная Fluent-композиция, короткие названия там, где текст мешает, и пояснения там, где пользователь принимает решение.

Проект технически является WinUI 3/Windows App SDK приложением, поэтому план опирается на WinUI и Fluent Design, а не на WPF-шаблоны.

## Источники и принципы

- WinUI Gallery: https://github.com/microsoft/WinUI-Gallery
  - Использовать как ориентир для структуры shell: TitleBar сверху, NavigationView слева, контентная область справа.
  - Брать подход "interactive companion": UI должен показывать состояние и давать действие рядом с контекстом.
- Windows app design: https://learn.microsoft.com/en-us/windows/apps/design/
  - Design principles: effortless, calm, personal, familiar, complete/coherent.
  - Guidelines: color, commanding, elevation, geometry, iconography, layout, materials, motion, navigation, typography, usability, widgets, writing.
- NavigationView guidance: https://learn.microsoft.com/en-us/windows/apps/develop/ui/controls/navigationview
  - Для нескольких верхнеуровневых страниц использовать NavigationView.
  - Для WinUI Gallery-паттерна держать TitleBar над NavigationView.
- Title bar design: https://learn.microsoft.com/en-us/windows/apps/design/basics/titlebar-design
  - Верхняя панель должна идентифицировать приложение, позволять перетаскивать окно и оставлять системные кнопки свернуть/развернуть/закрыть доступными и предсказуемыми.
- Writing style: https://learn.microsoft.com/en-us/windows/apps/design/style/writing-style
  - Текст короткий, активный, без лишних вводных. Полные пояснения только у рискованных или неочевидных настроек.

## Информационная архитектура

### Верхний уровень

1. Профили
   - Основной профиль прокрутки.
   - Дополнительные профили.
   - Создание профиля.
2. Исключения
   - Текущее активное приложение.
   - Ручное добавление исключения.
   - Автоматические правила для полноэкранного режима.
   - Список правил и поиск.
3. Настройки
   - Главный переключатель плавной прокрутки.
   - Автозапуск и поведение окна.
   - Тема, импорт, экспорт, сброс.

### Навигация

- Перевести NavigationView из Top в Left.
- Оставить 3 пункта верхнего уровня, потому что структура плоская и пользователь может открывать разделы в любом порядке.
- Использовать короткие подписи: "Профили", "Исключения", "Настройки".
- Добавить визуальную область бренда/статуса в панели слева, чтобы сразу было понятно, что делает приложение.

## Визуальная система

### Shell

- Окно: MicaBackdrop как базовый слой.
- Верхняя панель:
  - 48 epx высота для более удобной зоны перетаскивания.
  - Название приложения слева.
  - Короткий статус справа от названия или в правой части до системных кнопок.
  - Системные caption buttons остаются нативными.
- Левая навигация:
  - Expanded width около 240 epx.
  - Компактная ширина около 56 epx.
  - Иконки Segoe Fluent/WinUI SymbolIcon, подписи без длинных фраз.
- Контент:
  - Один основной ScrollViewer на странице.
  - Максимальная ширина контента, чтобы формы не растягивались слишком широко.
  - Отступы 24-32 epx, секции сгруппированы по смыслу.

### Цвет и материалы

- Не делать однотонную "синюю" или "фиолетовую" тему.
- Использовать системные ThemeResource-ки:
  - `ApplicationPageBackgroundThemeBrush`
  - `LayerFillColorDefaultBrush`
  - `CardBackgroundFillColorDefaultBrush`
  - `CardStrokeColorDefaultBrush`
  - `TextFillColorPrimaryBrush`
  - `TextFillColorSecondaryBrush`
- Accent использовать только для главного действия или активного состояния.

### Геометрия и слои

- Карточки: radius 8, stroke 1, без вложенных карточек.
- Повторяемые элементы списка остаются карточками/строками, но страницы не должны выглядеть как набор случайных блоков.
- Внутренние строки настроек должны иметь стабильную сетку: label/description слева, control справа.

### Типографика

- Segoe UI Variable через WinUI defaults.
- Заголовки страниц: 28/36 semibold.
- Заголовки секций: 20/28 semibold.
- Основной текст: 14/20.
- Подсказки: 12/16 или 14/20 вторичным цветом.
- Sentence case для UI-текста.

### Команды и UX-текст

- Кнопки с иконкой и коротким действием: "Добавить", "Выбрать", "Экспорт", "Импорт", "Сбросить".
- Не дублировать смысл рядом с очевидным контролом.
- Описания добавить к настройкам с последствиями:
  - Автозапуск.
  - Запуск в трее.
  - Закрытие в трей.
  - Полноэкранные окна.
  - Сброс настроек.
- Диалоги уже системные, тексты в них оставить понятными.

## Критерии приемки

- Слева есть стабильная навигация.
- Сверху есть нормальная панель окна с названием приложения, drag-областью и нативными кнопками свернуть/развернуть/закрыть.
- Каждый раздел можно понять за несколько секунд по заголовку, подписи и первому блоку.
- Настройки не выглядят как случайная стопка элементов: есть сетка, иерархия и одинаковые отступы.
- Текст не перегружает интерфейс и не исчезает там, где пользователь должен понять последствия действия.
- Сборка проходит без ошибок после каждого этапа.
